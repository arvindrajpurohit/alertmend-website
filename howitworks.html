How It Works/Use cases

Effortlessly Manage Incidents and Enhance Productivity with AlertMend

Node Failure Alert Recived

  • AlertMend listens to alerts indicating that a node has failed.
  • AlertMend processes the alert to understand the details of the node failure
  • AlertMend identifies which pods were running on the failed node
  • AlertMend triggers actions to reschedule the affected pods onto healthy nodes to restore service continuity.

Persistent Volume Claim (PVC) Disk Full Alert Received

  • AlertMend listens to alerts indicating that a Persistent Volume Claim (PVC) is full.
  • AlertMend processes the alert to identify the PVC and the associated pod or application affected by the full disk.
  • AlertMend checks for the largest files or data within the PVC to determine the cause of the disk full condition.
  • AlertMend triggers actions such as archiving or deleting unnecessary files, expanding the PVC, or notifying the relevant teams to take further action.

Pod Not Running State Alert Received

  • AlertMend listens to alerts indicating that a pod is not in the running state.
  • AlertMend checks the pod's logs and events to diagnose the issue (e.g., image pull errors, insufficient resources, failed readiness probes).
  • AlertMend verifies the status of the node hosting the pod and checks for any issues that might prevent the pod from running (e.g., node pressure, network failures).
  • AlertMend triggers actions to resolve common issues, such as restarting the pod, rescheduling it to a different node, or scaling up resources if required.

High JVM Usage Alert Received

  • AlertMend listens to alerts indicating high JVM usage for an application.
  • AlertMend processes the alert to identify the application and the specific pod or instance experiencing high JVM usage.
  • AlertMend triggers an action to take a JVM heap dump to capture the memory state of the application.
  • AlertMend securely moves the heap dump file to designated block storage for further analysis by the engineering team.

Payment Failure Alert Received

  • AlertMend listens to alerts indicating multiple payment failures on the payment gateway.
  • AlertMend checks the payment gateway logs for error messages or unusual patterns that might indicate the root cause (e.g., network issues, authentication errors).
  • AlertMend verifies the status of the bank's gateway or API to check for outages or service interruptions on the bank's side.
  • AlertMend triggers actions to retry the failed transactions, if applicable, or automatically switch to an alternative payment route if supported.
  • AlertMend notifies the relevant teams and stakeholders with detailed information on the payment failures, the findings, and the actions taken for resolution.

50x Error on Ingress Alert Received

  • AlertMend listens to alerts indicating a 50x error (e.g., 500, 502, 503) occurring on the ingress.
  • AlertMend processes the alert to identify the affected service, backend pods, or endpoints associated with the ingress.
  • AlertMend checks the ingress controller logs for error messages or anomalies that might indicate the root cause (e.g., misconfigurations, upstream failures, SSL issues).
  • AlertMend verifies the health of backend services and pods, checking for readiness and liveness probe failures, resource constraints, or application errors.
  • AlertMend triggers actions to restart affected pods, adjust ingress configurations, or reroute traffic to healthy pods or services if available.
  • AlertMend notifies the relevant teams with detailed information on the 50x errors, including the findings and actions taken to resolve the issue.
